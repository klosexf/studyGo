# 确认命题模块视觉复刻设计

## 目标

将训练流程第 2 步“确认命题”的 UI 样式和布局调整为与
`逻辑表达训练产品_UI效果图.html` 中对应模块一致。效果图是唯一视觉基准，
不增加或替换设计元素。

## 范围

- 复刻顶部五阶段导航在“确认命题”状态下的尺寸、间距、圆角、颜色和排版。
- 复刻 `STEP 2 · 训练命题` 标识、命题标题框及背景说明。
- 复刻黄色主问题卡，包括标题、任务说明、场景与难度、标签。
- 复刻“表达约束”和“评分重点”双列白色卡片。
- 保留现有命题提示面板和底部操作按钮，继续排列在上述内容之后。
- 保留现有动态命题数据、事件处理、状态流转和无障碍语义。

## 实现边界

- 样式通过确认命题阶段专用类名限定，避免改变设置训练、写初稿、诊断改写和结果复盘。
- `TopicView` 仅调整展示结构及专用类名，不改变组件参数和回调。
- `StageTabs` 保持共享逻辑；仅在确认命题阶段应用效果图要求的视觉状态。
- 不删除命题提示、返回、重新生成或开始写初稿功能。
- 不修改命题文案内容；所有标题、背景、问题、标签、约束和评分重点继续来自当前数据。

## 视觉规格

桌面端以用户提供的截图和效果图 HTML 为基准：

- 阶段导航为横向五列，已完成步骤为浅绿色，当前步骤为深蓝色，其余步骤为暖灰色。
- 页面主体使用纵向大间距布局。
- 命题标题置于浅色描边的大圆角容器内，使用超大黑色粗体。
- 背景说明位于标题框下方，使用中灰色正文。
- 主问题卡使用浅黄色背景、大圆角和宽松内边距。
- 标签使用浅灰色胶囊样式。
- 两张详情卡等宽双列、白色背景、大圆角，无额外装饰。
- 阴影、描边、字号、行高、间距及圆角直接依据效果图，不新增渐变或强调色。

## 响应式

- 桌面宽度优先保持效果图比例。
- 窄屏下阶段导航允许横向滚动。
- 窄屏下详情双列改为单列，文字和卡片按现有页面宽度收缩。
- 响应式只解决可用性，不改变颜色、层级和内容顺序。

## 验证

- 运行相关单元测试，确认训练流程与按钮行为未回归。
- 在本地浏览器中进入“确认命题”阶段，对照效果图检查导航、标题、卡片、
  标签和双列区域。
- 检查桌面视口和移动视口，确认无横向溢出和内容遮挡。
