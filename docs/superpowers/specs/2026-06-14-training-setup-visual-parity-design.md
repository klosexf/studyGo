# Training Setup Visual Parity Design

## Goal

将训练流程第 1 步按仓库内同源高保真设计稿 1:1 还原，修正当前页面在结构、尺寸、颜色、间距和选择状态上的偏差。

## Scope

- 在训练内容顶部增加“设置训练 / 第 1 步 / 5”和搜索框。
- 将五步进度条改为设计稿中的等宽胶囊式导航。
- 将第 1 步改为左右双卡布局：左侧场景与难度，右侧系统推荐。
- 保留现有场景、难度、训练目标、Provider 和生成命题逻辑。
- 推荐训练目标继续来自现有 `TrainingConfig.trainingGoal`，只改变呈现方式。
- 第 2 至第 5 步保持现有业务结构，仅共享新的顶部区域和进度条视觉。
- 小屏切换为单列，不能产生横向溢出。

## Visual Rules

- 页面背景使用接近白色的冷灰紫色，主文字为 `#20211f`。
- 顶部标题字号约 44px，副标题为灰色 18px。
- 搜索框高度约 72px，圆角 24px，无默认阴影。
- 进度容器为浅灰背景、约 72px 高；当前步骤为深蓝灰。
- 设置区两列等宽，卡片圆角约 32px。
- 左侧场景选项为竖向大胶囊，选中项使用浅鼠尾草绿色。
- 中等难度使用暖黄色。
- 推荐卡使用浅薰衣草色，CTA 使用高饱和紫色。

## Behavior

- 点击场景卡和难度按钮仍通过 `onChange` 更新现有配置。
- 点击“生成训练命题”仍调用 `onGenerate`。
- 搜索框在本次改动中仅作为设计稿中的页面级搜索入口，不改变训练流程数据。
- 可访问性语义保留：进度为 `nav` + `ol`，场景为单选控件，难度使用 `aria-pressed`。

## Verification

- 单元测试覆盖目标文案、默认状态、场景切换和 CTA。
- 运行 TypeScript、ESLint 和训练工作区测试。
- 在本地浏览器以桌面与移动视口检查尺寸、双列布局和横向溢出。
